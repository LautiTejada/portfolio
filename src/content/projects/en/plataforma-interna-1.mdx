> **Confidentiality note**: this project was developed for a company under a confidentiality agreement. Names, specific data, and screenshots have been anonymized. The description reflects the work performed without revealing protected information.

## Problem

The operations team generated weekly reports manually: they exported data from the database, processed it in Excel, and built charts by hand. The process took between 4 and 6 hours per week and was prone to human error.

There was no centralized dashboard. Each department consulted its own spreadsheets with outdated data.

## Solution

A reporting module was designed and integrated into the existing internal platform. The frontend consumes a REST API that consolidates data from multiple sources and presents it in interactive dashboards with filters by date, department, and key metrics.

Client-side export was implemented for PDF (using the native print API) and Excel (CSV generation with UTF-8 encoding).

## Technical Decisions

- **Recharts over D3**: Recharts offers declarative React components that integrate well with the existing design system. D3 would have given more control but at a significantly higher development cost for the chart types required (bar, line, pie).
- **React Query for data fetching**: automatic caching, polling every 30s for real-time data, and loading/error state handling without boilerplate.
- **Client-side export**: avoids the need for a file generation service on the backend. For PDF, `window.print()` is used with dedicated print styles. For Excel, a CSV is generated directly in the browser.
- **Tailwind CSS**: consistency with the rest of the platform and fast iteration on table and metric card design.

## Architecture

The module integrates as a section within the existing SPA:

- `/reports` — Main view with tabs by report type
- `/reports/dashboard` — Real-time dashboard with charts and KPIs
- `/reports/export` — Export configuration and generation
- `hooks/useReportData.ts` — Custom hook over React Query for filtered queries
- `components/charts/` — Reusable chart components wrapping Recharts

The backend API (out of scope for this project) exposes REST endpoints with pagination and filters. The frontend has no direct access to the database.

## Results

- Report generation time: from 4-6 hours/week to < 5 minutes.
- Adoption: 100% of the operations team uses the dashboard daily since the second week.
- Zero data errors reported in the first 3 months (vs. frequent errors in the manual process).
- Polling every 30s enables near real-time operational decision-making.

## What I Would Do Differently

I would implement WebSockets or Server-Sent Events instead of polling for real-time data — polling every 30s is functionally sufficient but generates unnecessary load on the API. I would also add integration tests for the chart components, which are difficult to test with simple unit tests due to their SVG dependency.
