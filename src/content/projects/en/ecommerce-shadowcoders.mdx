## Context

E-commerce project developed as a team under the name ShadowCoders. The goal was to build a complete e-commerce platform, from the product catalog to checkout with payments, including an admin panel for inventory management.

## Solution

A separated frontend/backend architecture was developed:

**Frontend**: SPA built with React 19, Vite, and TypeScript. Material UI (MUI 7) was used as the design system, Zustand for global state (cart, authentication), React Router DOM for navigation, and Axios for API communication.

**Backend**: REST API developed in Java with Spring Boot, handling business logic, data persistence, authentication, and authorization.

## Technical decisions

- **React 19 + Vite**: modern and fast development combo. Vite provides instant HMR and optimized builds. React 19 brings performance improvements.
- **Zustand over Redux**: simple global state without boilerplate. For an e-commerce with cart and auth, Zustand covers the needs without Redux Toolkit's complexity.
- **Material UI 7**: high-quality pre-built components that accelerated development of tables, forms, modals, and navigation.
- **Separated frontend/backend architecture**: allows each team to work independently. The frontend consumes documented REST endpoints.
- **Swiper for carousels**: product carousels with native touch support and smooth transitions.
- **SweetAlert2**: alerts and confirmations with better UX than native browser alerts.

## Architecture

**Frontend:**

```
dresscode-front/
├── src/
│   ├── components/     # UI components (ProductCard, Cart, Admin)
│   ├── pages/          # Main views (Home, Catalog, Checkout)
│   ├── store/          # Zustand stores (cart, auth, products)
│   ├── services/       # API calls with Axios
│   ├── types/          # TypeScript interfaces
│   └── utils/          # Helpers and constants
```

**Backend:**

```
├── src/main/java/
│   ├── controllers/    # REST endpoints
│   ├── services/       # Business logic
│   ├── models/         # JPA entities
│   ├── repositories/   # Data access
│   └── security/       # Authentication and authorization
```

## Teamwork

Development was collaborative: the frontend repository has 71 merged pull requests with contributions from multiple team members. We worked with a Git flow based on feature branches and code review via PRs.

## Results

- Functional e-commerce with catalog, search, persistent cart, and checkout.
- Admin panel for managing products, categories, and discounts.
- 71 merged PRs in the frontend with organized Git workflow.
- TypeScript (81.9% of code) ensuring type safety across the entire frontend.

## What I Would Do Differently

I would migrate to Next.js for SSR/SSG on catalog pages (better SEO and initial performance). I would also implement testing from the start — when developing as a team, tests prevent regressions when multiple people work on the same code. Finally, I would move `node_modules` out of the repository (currently committed in the dev branch).
