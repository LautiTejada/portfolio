> **Confidentiality note**: this is an internal Inncome application. Specific module names, financial data, and screenshots have been omitted. The description reflects the technical work performed without revealing protected information.

## Context

Beyond the public landing page, Inncome needed an internal web application where users could manage financial operations. The app required authentication, multiple views based on user role, and clear data presentation through dashboards and tables.

## Solution

The internal web application was developed as a separate project within the same repository, using Next.js with App Router and TypeScript. The app consumes an external REST API and presents data through interfaces organized by module.

Authentication with route protection, session management, and login state-based redirection were implemented.

## Technical decisions

- **Next.js App Router**: nested layouts to separate the authenticated app structure from public pages (login, registration). Server Components for initial data loading.
- **Strict TypeScript**: complete typing of API responses and component props to catch errors at development time.
- **Tailwind CSS**: design consistency with the landing page, reusing color and typography tokens.
- **Frontend/backend separation**: the app is purely frontend, consuming REST endpoints from an external backend. This allows both teams to work independently.

## Results

- Internal production application used by the Inncome team for daily operations.
- Intuitive navigation organized by modules with role-controlled access.
- Responsive interface enabling operation from both desktop and mobile devices.
