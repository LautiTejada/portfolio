> **Nota de confidencialidad**: este proyecto fue desarrollado para una empresa bajo acuerdo de confidencialidad. Los nombres, datos específicos y capturas han sido anonimizados. La descripción refleja el trabajo realizado sin revelar información protegida.

## Problema

El equipo de operaciones generaba reportes semanales de forma manual: exportaban datos de la base, los procesaban en Excel, y armaban gráficos a mano. El proceso tomaba entre 4 y 6 horas por semana y era propenso a errores humanos.

No existía un dashboard centralizado. Cada área consultaba sus propios spreadsheets con datos desactualizados.

## Solución

Se diseñó un módulo de reportes integrado a la plataforma interna existente. El frontend consume una API REST que consolida datos de múltiples fuentes y los presenta en dashboards interactivos con filtros por fecha, área y métricas clave.

Se implementó exportación client-side a PDF (usando la API nativa de print) y a Excel (generación de CSV con encoding UTF-8).

## Decisiones técnicas

- **Recharts sobre D3**: Recharts ofrece componentes declarativos de React que se integran bien con el design system existente. D3 hubiera dado más control pero con un costo de desarrollo significativamente mayor para el tipo de gráficos requeridos (barras, líneas, tortas).
- **React Query para data fetching**: caching automático, polling cada 30s para datos en tiempo real, y manejo de estados de carga/error sin boilerplate.
- **Exportación client-side**: evita la necesidad de un servicio de generación de archivos en el backend. Para PDF se usa `window.print()` con estilos de impresión dedicados. Para Excel, se genera un CSV directamente en el browser.
- **Tailwind CSS**: consistencia con el resto de la plataforma y velocidad de iteración en el diseño de tablas y cards de métricas.

## Arquitectura

El módulo se integra como una sección dentro de la SPA existente:

- `/reports` — Vista principal con tabs por tipo de reporte
- `/reports/dashboard` — Dashboard en tiempo real con gráficos y KPIs
- `/reports/export` — Configuración y generación de exportaciones
- `hooks/useReportData.ts` — Hook custom sobre React Query para queries con filtros
- `components/charts/` — Componentes de gráficos reutilizables wrapping Recharts

La API backend (fuera del alcance de este proyecto) expone endpoints REST con paginación y filtros. El frontend no tiene acceso directo a la base de datos.

## Resultados

- Tiempo de generación de reportes: de 4-6 horas/semana a < 5 minutos.
- Adopción: 100% del equipo de operaciones usa el dashboard diariamente desde la segunda semana.
- Cero errores de datos reportados en los primeros 3 meses (vs. errores frecuentes en el proceso manual).
- El polling cada 30s permite tomar decisiones operativas con datos casi en tiempo real.

## Qué haría diferente

Implementaría WebSockets o Server-Sent Events en lugar de polling para los datos en tiempo real — el polling cada 30s es suficiente funcionalmente pero genera carga innecesaria en la API. También agregaría tests de integración para los componentes de gráficos, que al depender de SVG son difíciles de testear con unit tests simples.
